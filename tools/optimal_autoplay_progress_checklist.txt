The following decisions were changed or added in this batch. Each can be reverted by inspecting
the trace reason fields in autoplay_scenarios.json or the DECISION_TRACES output from quicktest.

1. Gift wrap gate: threshold changed from dealer_happiness < 90 to dealer_happiness < 78
   (reason: "gift_wrap_happiness_gate" in event_policy.yes_no)
   Also: emergency check added - now declines wrap when health < 45, sanity < 20, or no car
   Old behavior: gift_wrap_for_dealer_happiness (if balance >= 20 and happiness < 90)
   Revert: restore in quicktest.py _decide_yes_no and event_policy.py choose_event_yes_no

2. Workbench crafting: bot now engages crafting at workbench when useful recipes are available
   Old behavior: always returned "5" (Pack Up) - never crafted anything
   New behavior: _choose_workbench_menu selects "1" (Craft Something) when best recipe priority >= 60
   Priority list is in config.py CRAFTING_RECIPE_PRIORITIES
   Revert: restore "return '5'" in quicktest.py _decide_raw_input workbench block

3. Millionaire afternoon: explicit routing to mechanic ending or airport
   Old behavior: only matched "drive to the airport" text in a very specific condition
   New behavior: _choose_millionaire_afternoon prefers chosen mechanic, then airport, then fallback
   Trace reason: "millionaire_mechanic_ending:{mechanic}" or "millionaire_airport_ending"
   Revert: restore original text-match logic in quicktest.py _decide_raw_input

4. Companion hungry gate: now uses companion_runaway_risk_count from game state
   Old behavior: chose "3" (buy food) if balance >= 20 else "4" (skip)
   New behavior: also chose "3" when companion_runaway_risk_count > 0 or companion_low_happiness_count > 0
   Trace reason: "hungry_companions_gate" in event_policy.py
   Revert: restore simple balance check in event_policy.py choose_event_inline_choice
