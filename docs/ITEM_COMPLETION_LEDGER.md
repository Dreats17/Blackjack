# Item Completion Ledger

> Per-item acceptance tracker for deciding when item work is actually done.
> Use this alongside `ITEM_PLANNING_REMAINING.md`.

---

## How To Use This File

- `Current Role` describes what the item does today at a high level.
- `Coverage Target` describes what counts as done for interconnectivity.
- `Status` should be one of: `done`, `partial`, `under-covered`, `optional`, or `cut`.
- `Notes` should point to the missing hooks or explain why the item is intentionally lightweight.

---

## Completion Standard

- `done`: the item meets the acceptance criteria in `ITEM_PLANNING_REMAINING.md`.
- `partial`: the item has some meaningful hooks, but still needs more coverage.
- `under-covered`: the item technically exists but still feels isolated.
- `optional`: the item depends on a system that may be cut.
- `cut`: deliberately not being expanded further.
